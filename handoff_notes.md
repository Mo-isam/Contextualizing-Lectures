## 🚀 What We Accomplished in This Session

We successfully completed a major architectural decoupling and component refactoring, resolving critical design bottlenecks:

1. **Decoupled Backend Pipeline (`core/pipeline.py` & `server.py`)**:
   - Extracted the monolithic execution pipeline out of the `websocket_process_stream` router in `server.py` and placed it inside a dedicated `PipelineJob` class in `core/pipeline.py`.
   - Preserved lazy ML loading inside the pipeline execution thread to keep boot-up RAM footprint low.
   - Switched from pseudocancellation (a boolean flag) to a thread-safe `threading.Event` checked periodically during runtime. The backend thread now detects disconnects immediately and terminates early, preventing CPU/RAM leaks.
   - Replaced manual path string manipulation with robust cross-platform `pathlib.Path` objects.

2. **Unified API Service Layer & Port Cleanup (`api.ts` & Views)**:
   - Created a centralized service client `frontend/src/services/api.ts` that encapsulates HTTP endpoints, multipart uploads, and WebSocket url builders.
   - Removed all hardcoded port `8000` bindings (such as `http://localhost:8000` or direct WS port mappings) across views (`App.tsx`, `LibraryView.tsx`, `UploadView.tsx`, `ProcessingView.tsx`, `SettingsModal.tsx`, `StudioView.tsx`).
   - WS endpoints now dynamically resolve relative to the hosting origin, enabling standard Vite proxy routing in development.

3. **Decomposed AudioPlayer Architecture (`AudioPlayer.tsx` & Utilities)**:
   - Moved slide overlap boundary computations, active playhead lookups, and duration clock string formatters out of the visual components and into a pure helper module `frontend/src/utils/boundaries.ts`.
   - Encapsulated WaveSurfer.js instantiation, ResizeObserver resize-rendering buffers, playback control states, and media cleanup inside a custom React hook `frontend/src/hooks/useWaveSurfer.ts`.
   - Decomposed the timeline segment stripe and segment jumps row into sub-components (`SlideTimeline.tsx` and `SlideJumpPills.tsx`).
   - Simplified the orchestrator component `AudioPlayer.tsx` from 500+ lines of monolithic logic to a readable ~170 line coordinator.

4. **Bug Fix: Infinite Seek Loop on Play/Pause**:
   - Fixed a bug where pausing the player after clicking a slide note's "Play at" button would trigger an infinite loop that snapped the playhead back to the initial seek time.
   - Added a `useRef` reference guard `lastProcessedSeekTo` to guarantee that seek requests are processed exactly once. Removed reactive state `isPlaying` from hook dependencies to prevent re-evaluation loops.

5. **Docs, Version Control & Git push**:
   - Updated `README.md` to map out the new folder structure and revised running instructions.
   - Committed changes across 7 clean atomic commits and pushed them to `feature/decoupled-architecture`.
